# Brief: Telegram Mini App — Секретарь для записи клиентов

> Ниша: ремонт бадминтонных ракеток  
> Версия: 1.0 (MVP)  
> Язык: русский  
> Дизайн: нативный Telegram (themeParams, MainButton, BackButton)  
> Цель: клиент записывается на консультацию за ≤ 3 минуты

---

## Архитектура навигации

```
[Старт]
   └─→ Экран 0: Приветствие
            └─→ Экран 1: Выбор услуги
                     └─→ Экран 2: Описание проблемы
                              └─→ Экран 3: Выбор даты
                                       └─→ Экран 4: Выбор времени
                                                └─→ Экран 5: Контакт (телефон)
                                                         └─→ Экран 6: Подтверждение
                                                                  └─→ Экран 7: Успех
                                                                           └─→ Экран 8: Мои заявки
                                                                                    └─→ Экран 9: Детали заявки
```

**BackButton** доступен на экранах 1–6 (возврат на предыдущий шаг без потери данных).

## Поток мастера (уведомления в Telegram-бот)

При каждой новой заявке мастер получает сообщение в личку:

```
📬 Новая заявка №42

👤 Аркадий Марков
📞 +7 900 123-45-67
🏸 Натяжка струны
🗓 вт, 8 апреля · 14:00

Проблема:
Струна лопнула во время игры,
нужна натяжка ~10кг

[✅ Принять]  [❌ Отклонить]
```

Inline-кнопки меняют статус заявки + уведомляют клиента.

## Технический стек

| Слой | Технология | Обоснование |
|------|------------|-------------|
| Frontend | React + Vite | Быстрая сборка, code splitting |
| UI-kit | `@telegram-apps/telegram-ui` | Нативные Telegram-компоненты |
| Backend | Python + FastAPI | Уже начат, добавить REST |
| База клиентов | Google Sheets (gspread) | Уже реализовано |
| Аутентификация | Telegram `initData` verify | Нет паролей, безопасно |
| Деплой | Vercel (frontend) + Railway (backend) | HTTPS из коробки, бесплатно |
| Уведомления | python-telegram-bot (уже есть) | Уже реализовано |

## Чего НЕ будет в первой версии

### Намеренно исключено (упростить и ускорить запуск)

| Функция | Причина исключения | Версия |
|---------|--------------------|--------|
| Онлайн-оплата / предоплата | Сложность интеграции провайдера | v2 |
| Реальная синхронизация слотов | Нужен бэкенд-календарь | v2 |
| Отмена/перенос клиентом | Через звонок мастеру пока достаточно | v2 |
| Напоминания за 24ч | Cron-задача, усложняет деплой | v2 |
| Галерея работ / фото | Контент-менеджмент, не нужен для MVP | v2 |
| Отзывы клиентов | Нет достаточной базы для старта | v3 |
| Чат с мастером | Telegram уже есть для этого | v3 |
| Мультиязычность | Только русский для целевой аудитории | v3 |
| Программа лояльности | Преждевременная сложность | v3 |
| Несколько мастеров | Продукт для одного мастера | v3 |

### Принципы MVP

1. **Один поток** — запись идёт только по одному сценарию
2. **Без авторизации** — Telegram-пользователь = авторизация
3. **Без оплаты** — только заявка, деньги обсуждаются при встрече
4. **Без реального календаря** — мастер видит заявку и сам подтверждает
5. **Без CMS** — контент (фото, описание) меняется только в коде

## Критерии готовности v1

- [ ] Клиент проходит весь флоу за ≤ 3 минуты
- [ ] Мастер получает уведомление в Telegram в течение 30 сек
- [ ] Заявка появляется в Google Sheets
- [ ] Mini App открывается по кнопке в боте (не ссылка)
- [ ] Работает на iOS и Android
- [ ] Нативные цвета Telegram (тёмная + светлая тема)
- [ ] Все кнопки ≥ 44px
- [ ] Нет белого экрана при загрузке (skeleton)
- [ ] HTTPS деплой
- [ ] `initData` проверяется на бэкенде
